## 技術構成
- 静的サイト。`npm run build`（`scripts/build.js`）が`data/*.csv`と`content/articles/*.md`を読み込み、`index.html`・`spots/{id}/`・`events/{id}/`・`articles/{slug}/`・`spots.json`・`events.json`・`sitemap.xml`・`robots.txt`を生成
- GitHub Actions（`.github/workflows/build.yml`）が**毎日6:00 JST自動ビルド＋手動実行(workflow_dispatch)＋data/content/scripts変更時のpushトリガー**でビルド→差分あればbotコミット→push（GitHub Pagesへ自動反映）
  - ⚠️ ワークフローファイルの初回pushには`gh auth refresh -s workflow`でのスコープ追加が必要だった（対応済み、今後は不要）
- 本番データソースは将来的にGoogleスプレッドシート（CSV公開URL）を想定。`data/source-config.json`に`spotsCsvUrl`/`eventsCsvUrl`/`reviewsCsvUrl`を設定すればそちらを優先（**まだ未設定、現在はローカルの`data/*.csv`で代替中**）
- npm依存: `csv-parse`（CSV解析）、`gray-matter`（Markdownフロントマター解析）、`marked`（Markdown→HTML変換）。フレームワークは使わず素のNode.jsスクリプト
- 地図: Leaflet + CARTO Voyagerタイル（APIキー不要）。天気: Open-Meteo（APIキー不要）。地域検索・座標検索: 国土地理院(GSI)住所検索API（APIキー不要）
- ローカル動作確認は`npx http-server`を手動起動＋curlで実施。**Claude Preview MCPツールが本セッション中`.claude/launch.json`を認識できない不具合が継続**（原因不明、ファイル自体は正常）

## データモデル
- `data/spots.csv`: 常設スポット。列は id/name/address/lat/lng/category/indoor/shade/water/photo_url/gallery/description/fee/parking/hours/official_url/amenities/target_age/duration/recommended_time/last_verified/added_date
  - `gallery`は`|`区切りで複数枚の追加写真パスを持てる（メイン写真=photo_url、それ以外=gallery）
  - `amenities`も`|`区切り（トイレ/オムツ替え台/授乳室/ミルク用のお湯/自販機/水飲み場/駐輪場/徒歩圏にコンビニ・ドラッグストア）
- `data/events.csv`: 期間限定イベント。`related_spot_id`が空なら独立イベント（期間中のみ地図・新着に表示）、値があればスポット紐付きキャンペーン（別ピンにせず対象スポットのカード/地図ポップアップ/詳細ページにバッジ表示）
- `data/reviews.csv`: 口コミ。`spot_id`で紐付け。**承認なしで自動公開する方針**（Googleフォーム経由を想定、まだフォーム自体は未作成）
- `content/articles/*.md`: まとめ記事。フロントマター（title/description/related_spots/published_date）+本文Markdown。**日付は必ずクォートすること**（`"2026-07-21"`）。YAMLの仕様上クォートなしだと自動的にDateオブジェクトに変換されるバグに一度当たった（build.js側で防御的に文字列化する処理は入れたが、書く側でもクォート推奨）

## 実装済み機能一覧
- 天気バナー（Open-Meteo）: 最高気温33℃以上で「暑い日」→屋内・日陰・水遊びをおすすめ。降水確率50%以上で「雨の日」→屋内のみおすすめ（雨判定が優先、両方該当時も雨側のロジックのみ適用）
- フィルター（屋内/日陰/水遊びチップ＋カテゴリ選択、OR条件で絞り込み）
- 「現在地から探す」: 位置情報取得→現在地マーカー表示→スポット一覧を距離順に並び替え
- 「おまかせで今日行く場所を決める」: 天気条件と現在地（あれば）から1件を自動提案、再提案（違う候補）にも対応
- 地域検索: 駅名・住所を入力→GSI住所検索APIで地図を移動（完全一致優先のロジック）
- 口コミ機能の土台: スポットページに口コミ一覧表示、投稿リンク（Googleフォーム想定、現状ダミーURL）
- AI要約（ページ全体要約）: `ANTHROPIC_API_KEY`が環境変数にあれば、スポットの基本情報＋口コミを総合して2〜4文の要約を生成（`scripts/build.js`の`summarizeSpotPage`）。**未設定の間は何もせず優雅にスキップ**（現状キー未設定でこの機能は非活性）
- イベント/キャンペーンの期間表示（上記データモデル参照）
- まとめ記事（Markdown、SEO用）。第1弾「雨の日でも遊べる！宝塚市周辺の屋内子供スポット3選」公開済み
- スポットの写真ギャラリー機能（複数枚対応）
